# The Engagement Workflow

This guide is the connective tissue of the pack. The 32 skills are individually useful, but
real consulting value comes from **chaining them into an engagement** and holding every output
to a standard that separates genuine executive work product from confident-sounding filler.

Read this before running a multi-skill engagement. Pair it with **[DESIGN-GUIDE.md](DESIGN-GUIDE.md)**
when you turn analysis into exhibits.

---

## Part 1 — The shape of an engagement

Top-tier strategy work follows a recognizable arc. The skills map onto it:

| Phase | Question it answers | Lead skills |
|---|---|---|
| **0. Frame** | What are we really solving, and how will we solve it? | `problem-structuring`, `situation-assessment`, `growth-barriers` |
| **1. Analyze** | What is true? Where's the value? | `market-mapping`, `competitive-intel`, `customer-segmentation`, `profit-pool-analysis`, `unit-economics`, `margin-bridge`, `financial-modeling`, `valuation` |
| **2. Choose** | What should we do? | `strategic-options`, `scenario-planning`, `pricing-strategy`, `go-to-market`, `portfolio-review`, `capital-allocation`, `commercial-due-diligence`, `synergy-valuation` |
| **3. Stress-test** | What would have to be true? What breaks it? | `assumption-audit`, `war-gaming` |
| **4. Build** | How do we deliver it? | `operating-model-design`, `cost-transformation`, `initiative-prioritizer`, `transformation-roadmap` |
| **5. Govern** | How do we capture and protect the value? | `risk-mitigation`, `kpi-architect`, `value-realization` |
| **6. Communicate** | How do we get a yes? | `stakeholder-alignment`, `narrative-builder`, `decision-memo` |

**The golden rule of sequencing: never skip Phase 0.** 80% of bad strategy work is a beautiful
answer to the wrong question. Frame first; the frame tells you which analysis skills to even open.

### The hypothesis-driven spine
Don't run every skill. After `problem-structuring`, you have a MECE issue tree with a hypothesis
on each branch. **Only run the analysis skills that confirm or kill a load-bearing hypothesis.**
A focused engagement might touch 6 skills; a comprehensive one, 15. Running all 32 by default is
motion without judgment.

## Part 2 — Engagement recipes

Common situations and the skill chains that fit them. Adapt — these are starting lines, not rails.

**Corporate growth strategy**
`problem-structuring` → `situation-assessment` → `growth-barriers` → `market-mapping` + `customer-segmentation` + `profit-pool-analysis` → `strategic-options` (+ `scenario-planning`) → `business-case-builder` → `assumption-audit` → `transformation-roadmap` → `narrative-builder` → `decision-memo`

**Turnaround / margin recovery**
`situation-assessment` → `problem-structuring` → `margin-bridge` → `cost-transformation` + `pricing-strategy` → `initiative-prioritizer` → `transformation-roadmap` → `value-realization` → `kpi-architect` → `decision-memo`

**M&A / acquisition evaluation**
`problem-structuring` (deal thesis) → `commercial-due-diligence` → `market-mapping` + `competitive-intel` → `financial-modeling` → `valuation` → `synergy-valuation` → `assumption-audit` → `risk-mitigation` → `decision-memo` (IC paper)

**Market / segment entry**
`market-mapping` → `customer-segmentation` → `competitive-intel` → `pricing-strategy` + `unit-economics` → `go-to-market` → `business-case-builder` → `war-gaming` → `transformation-roadmap`

**Cost program**
`margin-bridge` → `cost-transformation` (ZBB / should-cost / spans-and-layers) → `operating-model-design` → `initiative-prioritizer` → `value-realization` → `stakeholder-alignment` → `decision-memo`

**Portfolio / capital strategy**
`portfolio-review` → `valuation` (per unit) → `capital-allocation` → `strategic-options` (per unit) → `decision-memo`

**Board strategy review**
`situation-assessment` → `scenario-planning` → `strategic-options` → `risk-mitigation` → `narrative-builder` → `stakeholder-alignment` → `decision-memo`

## Part 3 — How outputs hand off

Each skill produces an artifact that becomes a named input to the next. Honor the chain:

- `problem-structuring` produces the **issue tree + workplan** → tells every analysis skill what to prove.
- `market-mapping` produces the **SOM / revenue ramp** → becomes the top line in `business-case-builder` / `financial-modeling`.
- `customer-segmentation` produces **segment WTP & cost-to-serve** → feeds `pricing-strategy` and `go-to-market`.
- `financial-modeling` produces the **FCF stream** → discounted by `valuation`.
- `strategic-options` produces the **recommended option** → economics via `business-case-builder`, stress-test via `assumption-audit` / `war-gaming`.
- `assumption-audit` / `war-gaming` produce **failure modes** → become the register in `risk-mitigation` and the triggers in `kpi-architect`.
- `business-case-builder` / `synergy-valuation` produce **promised benefits** → tracked by `value-realization`.
- Everything converges into `narrative-builder` → `decision-memo`, pre-wired by `stakeholder-alignment`.

**Carry the numbers forward, not just the conclusions.** If `market-mapping` said "$30M SOM, swinging
on adoption," the business case must use that exact number and that exact swing — not a fresh, untethered
guess. Consistency across artifacts is a hallmark of real work and the first thing weak output gets wrong.

## Part 4 — The quality standard

Weak strategy output has a specific signature: fluent, structured, confident — and empty. It uses
the vocabulary of rigor without the substance. Every output in this pack must clear the bar below.
**When you finish a deliverable, run it against this checklist before showing anyone.**

### The seven tests

1. **The specificity test.** Could this sentence appear in a deck for *any* company in *any* industry?
   If yes, delete or replace it. "Leverage synergies to drive growth" is filler. "Consolidate the two
   overlapping ERP instances to save $14M by FY26" is work.

2. **The sourcing test.** Every number and factual claim is either sourced, derived from something
   sourced, or explicitly tagged `[assumption]` / `[estimate]` / `[to be confirmed]`. A confident number
   with no provenance is the single biggest tell of weak work. *Never invent data to fill a template.*

3. **The "so-what" test.** Every finding, chart, and section ends in a decision implication. If you can't
   write the "so what," the content doesn't belong. A fact base with no implications is a data dump.

4. **The falsifiability test.** Recommendations name what would change them; assumptions name the test that
   would kill them; forecasts name the swing drivers. Conclusions that can't be wrong can't be trusted.

5. **The exposed-logic test.** Models are driver-based and a reviewer can change one input and see the
   answer move. No hard-coded growth rates, no black-box outputs. Show the build.

6. **The honest-range test.** Uncertainty is expressed as a range with its driver ("$440–510M, swinging on
   WACC and terminal growth"), never as false-precision ("$847.3M"). False precision signals the author
   doesn't understand their own uncertainty.

7. **The "would a partner cut it?" test.** Read it as a skeptical senior partner / CFO. Where would they
   write "so what?", "source?", "this is generic", or "you've buried the answer"? Fix those before they do.

### Language discipline

Strike these on sight — they signal empty content:
- Throat-clearing: *"In today's rapidly evolving landscape…", "It is important to note that…"*
- Empty intensifiers: *"robust", "world-class", "cutting-edge", "holistic", "synergistic", "best-in-class"* (unless quantified).
- Hedge stacking: *"may potentially could possibly help to…"* — commit or cut.
- Verb-free arguments: section headers like *"Market", "Financials"* posing as insight. Every key line is an **assertion with a verb and a point of view**.
- Bullet sprawl: 12 undifferentiated bullets where 3 ranked ones would do.

### Write the way partners write
- **Answer first.** Lead with the conclusion; supporting logic follows (Pyramid Principle / `narrative-builder`).
- **One idea per sentence.** Short, declarative, specific.
- **Quantify or cut.** A claim worth making is worth a number.
- **MECE.** No overlaps, no gaps — in arguments, in exhibits, in lists.
- **Earn every page.** If a slide/section doesn't change a decision, it's deleted, not demoted to an appendix you secretly hope no one reads (put genuine backup in the appendix; don't pad).

## Part 5 — Working with Claude on this

The same rigor applies to *how* you run the work, not just the output:

- **Feed real data.** These skills are multipliers on inputs. With no data, Claude will produce a
  plausible *structure* — useful as a template, but mark it clearly as illustrative and replace the
  numbers. Don't ship illustrative numbers as findings.
- **Make Claude show provenance.** Ask it to tag each number as sourced / estimated / assumed. If it
  can't, that's a gap to close, not a number to trust.
- **Force the counter-case.** After any recommendation, run `assumption-audit` or ask for the strongest
  argument *against*. One-eyed advocacy is weak analysis with good posture.
- **Iterate on the hypothesis, not the formatting.** Spend effort sharpening the answer and its evidence,
  not polishing prose around a weak conclusion.
- **Demand reconciliation.** When numbers appear in multiple artifacts, ask Claude to confirm they tie
  out. Inconsistent figures across a deck destroy credibility instantly.
- **Right-size the rigor.** A 2-day diagnostic and a board-level capital decision need different depth.
  Match the analytical investment to the reversibility and size of the decision.

## Part 6 — Quality gates by phase

Don't advance a phase until it clears its gate:

- **After Frame:** Can you state the decision question in one sentence and show a MECE issue tree with a hypothesis on each branch? If >50% of the tree is unknown, the deliverable is a workplan, not an answer.
- **After Analyze:** Does every finding have level + trend + benchmark + a "so what"? Do the numbers reconcile across analyses?
- **After Choose:** Is there a real option set including a base case, weighted criteria set *before* scoring, and a *conditional* recommendation?
- **After Stress-test:** Do you know what would have to be true and what breaks the plan — with pre-committed triggers?
- **After Build:** Does every workstream have one accountable owner, exit criteria (not dates), and a credible first-90-days win?
- **After Govern:** Is value baselined, net, and gated; are risks owned with leading indicators; do metrics resist gaming?
- **After Communicate:** Is it answer-first, one page where it counts, pre-wired with stakeholders, and ready for the hostile question?

If a gate fails, the honest move is to go back — not to paper over it with confident prose. That choice,
made consistently, is the whole difference between MBB-grade output and confident filler.
